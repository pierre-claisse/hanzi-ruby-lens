# Implementation Plan: Palette Fonts

**Branch**: `020-palette-fonts` | **Date**: 2026-02-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/020-palette-fonts/spec.md`

## Summary

Each of the six color palettes gains a dedicated CJK font that covers
the entire interface (Chinese text, pinyin annotations, and UI elements).
When the user switches palettes, both colors and font change. Fonts are
bundled via npm packages and switched through a single CSS custom
property (`--font-palette`), extending the existing palette mechanism. No
new persistence is needed — the font is derived from the palette ID.

### Key Implementation Details

**CSS Variable Switching** (existing pattern extended):

```css
[data-palette="vermillion-scroll"] {
  --color-background: 254 252 243;
  --color-text: 45 45 45;
  --color-accent: 200 75 49;
  --font-palette: "Cactus Classical Serif";
}
```

**Tailwind Config** (single family with generic sans-serif fallback):

```ts
fontFamily: {
  palette: ['var(--font-palette)', 'sans-serif'],
},
```

**Font Family Names** (from each npm package's @font-face):

| Palette           | `--font-palette` value          |
|-------------------|---------------------------------|
| vermillion-scroll | `"Cactus Classical Serif"`      |
| jade-garden       | `"LXGW WenKai TC"`              |
| indigo-silk       | `"Chiron Hei HK Variable"`      |
| plum-blossom      | `"Huninn"`                      |
| golden-pavilion   | `"Chiron Sung HK WS"`           |
| ink-wash          | `"Chocolate Classical Sans"`    |

**Important**: These font-family names were verified during implementation
by inspecting each package's generated CSS. Note: Chiron Sung HK uses
`"Chiron Sung HK WS"` (not `"Chiron Sung HK VF"` as originally planned).
Cactus Classical Serif and Chocolate Classical Sans are by Moonlit Owen,
using orthodox Traditional Chinese glyph forms.
