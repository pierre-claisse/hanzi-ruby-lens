# Implementation Plan: Library Tags

**Branch**: `029-library-tags` | **Date**: 2026-02-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/029-library-tags/spec.md`

## Summary

Add a Tag system to the library: Tag CRUD via a modal dialog, tag assignment via right-click context menu, multiselect tag filtering in the title bar, and sort-by-date toggle. Tags have a label and a color (from a predefined palette), stored in SQLite with a many-to-many junction table. Backend filtering via parameterized SQL.

## Technical Context

**Language/Version**: Rust stable (backend), TypeScript 5.9 (frontend)
**Primary Dependencies**: Tauri 2, React 19, rusqlite 0.38, lucide-react, Tailwind CSS 3.4
**Storage**: SQLite (local, `%APPDATA%\com.hanzirubylens.app\hanzi-ruby-lens.db`)
**Testing**: Vitest + @testing-library/react (frontend), cargo test (Rust) — all in Docker
**Target Platform**: Windows (Tauri desktop)
**Project Type**: Desktop (Tauri: Rust backend + React frontend)
**Performance Goals**: Tag operations < 5s, filtering instant for ≤ 500 Texts (SC-001–SC-004)
**Constraints**: Offline-first, all data in local SQLite, no network dependency
**Scale/Scope**: Up to 500 Texts, up to 50 Tags

| I. Content-First Design | PASS | Tags are small colored chips on cards; modal dialog isolates CRUD away from content. Filter dropdown is unobtrusive in title bar center. |

| VI. Docker-Only Execution | PASS | All tests and builds run in Docker. |
